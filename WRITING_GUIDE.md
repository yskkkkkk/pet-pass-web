# Pet-Pass Dev Blog 작성 가이드

## 시리즈 분류

| 시리즈 태그 | 예시 포스트 |
|---|---|
| 바이브 코딩 개발기 · 이슈 #N | post2, post3, post6, post7, post8 |
| 기획 & 디자인 · #N | post4, post5 |
| 바이브 코딩 개발기 · 시작하며 | post1 |

## lesson-box 제목 규칙

포스트 말미의 정리 박스(`<div class="lesson-box">`) 제목은 시리즈에 따라 아래 둘 중 하나만 사용한다.

| 시리즈 | h3 텍스트 |
|---|---|
| 이슈 (#N) | `이 문제에서 배운 것` |
| 기획 & 디자인 | `이 경험에서 배운 것` |
| 시작하며 (post1) | 자유 (lesson-box 없음) |

"이 경험에서 가져간 것", "이 이슈에서 배운 것", "이 글에서 배운 것" 등의 변형은 사용하지 않는다.

## index.html 카드 날짜

`<div class="blog-post-date">` 안에 날짜(연. 월. 일)는 적지 않는다.
시리즈 태그와 이슈 번호만 표기한다.

```html
<!-- 올바른 예 -->
<div class="blog-post-date">바이브 코딩 개발기 · 이슈 #4 (2)</div>

<!-- 사용하지 않음 -->
<div class="blog-post-date">2026. 04. 27 · 바이브 코딩 개발기 · 이슈 #4 (2)</div>
```

## lesson-box CSS

`blog_post.css`에 공통 정의되어 있다. 포스트별 인라인 override 금지.

- 좌측 border: `border-left: 4px solid var(--primary)`
- 제목(h3): `font-weight: 800`, `color: var(--text-primary)`

## 어투 지침

목표: "틀에 맞춰 찍어낸 보고서"가 아니라 "차분하게 성숙한 글쓴이 본인"의 목소리를 유지한다.

### 핵심 원칙

**나이와 경력에 맞는 자연스러운 단어**
과한 구어는 정제하되, AI 특유의 딱딱함이 아니라 차분한 직무 성숙도로 다듬는다.

**말줄임표(`...`)로 남기는 여운**
완벽한 정답을 내리는 강사처럼 쓰지 않는다. 고민의 한계나 아쉬움을 솔직히 인정하며 말끝에 `...`를 의도적으로 남긴다.

**독백·일기 형식의 서사적 도입**
기술 정의로 시작하지 않는다. 최근 겪은 실무 경험이나 문제를 툭 던지는 독백으로 연다.

**자기 객관화 표현 허용**
"물론 나도 ~하긴 했지만...", "사실 잘 모르고 쓴 게 맞다..." 등 솔직한 한계 인정 표현은 오히려 글에 생기를 준다.

### 구조·마크다운 규칙

- `### 🤦 문제 발생`, `### 🔍 원인 파악`, `### ✅ 해결` 같은 이모지 데코 제목 금지
- 한 문단에 서너 문장씩 우겨넣지 않는다. 1~2문장 단위로 나눈다.
- 볼드(`**`)는 코드명·수치·고유명사 위주로만 쓴다. 감정·판단을 담은 서술에는 쓰지 않는다.

### 금지 표현

**과잉 서사형 도입부**

| 금지 예 |
|---|
| "이 이야기는 아주 단순한 질문에서 시작됐다" |
| "시작은 사소했다" |
| "~이(가) 발단이었다" |
| "그게 이 이야기의 시작이었다" |

대신 상황을 그냥 툭 던진다 — "며칠 전 ~하다가 ~가 발생했다."

**AI식 문장 자동화 패턴**

| 금지 예 | 대체 |
|---|---|
| "소중한 시간이었다" | 삭제하거나 구체적 감상으로 |
| "기대 이상이었다" | 뭐가 구체적으로 좋았는지 1문장 |
| "유익한 프로젝트였다" | 삭제하거나 무심한 단상형으로 |
| "~를 깨달은 대목이다" | "~인가 보다." 정도로 무심하게 |
| "앞으로 더 많아질 것 같다" | 삭제 — 트렌드 예측은 쓰지 않는다 |

**분류·전환 문장 (예고·요약성 메타 코멘트)**

| 금지 예 |
|---|
| "이 경험에서 배운 건 생각보다 범위가 넓었다" |
| "두 케이스 모두 ~라는 게 공통점입니다" |
| "한 가지는 분명하게 느꼈습니다" |

### 결말부 패턴

**절대 금지 — AI식 억지 교훈·병렬 대비 결론**
- "결국 A와 B는 맞닿아 있다"
- "배운 점은 ~이었다" (단독 문장)
- 억지로 웅장하거나 감동적으로 끝맺으려 하지 않는다.

**승인된 결말 패턴**

1. **무심한 단상형** — 해결 후 가볍게 느낌만 던지고 끝
2. **기술적 사실 종결형** — 기술 요약만 나열하고 끝
3. **다음 스텝 / 셀프 피드백형** — 미완의 한계나 다음 시도할 것을 메모처럼 남김
4. **소박한 다짐형** — 일기장에 적듯 담담한 향후 계획

**직접 써야 하는 패턴 (AI가 흉내 내지 않음)**
- 일상·잡담형 추신(`ps.`)
- 개인적인 결이 있는 마무리
