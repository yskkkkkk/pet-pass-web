# 🔬 AI Lab 가이드 및 지침 (AI Instructions)

이 디렉토리는 `Google AI Studio`에서 실험하고 검증한 로직, 스크립트, 그리고 프롬프트를 보관하는 '프로젝트 지식 창고'입니다. 모든 AI 에이전트(Jules, Antigravity 등)는 작업을 수행하기 전 반드시 이 가이드를 숙지해야 합니다.

## 🎯 목적
1. **비용 효율화**: 검증된 로직을 재사용하여 불필요한 외부 검색 및 토큰 소모를 방지함.
2. **정확도 향상**: 사용자가 AI Studio에서 직접 테스트하고 승인한 로직을 최우선으로 적용함.
3. **지식 전달**: 개별 에이전트가 파악하기 어려운 복잡한 변환 로직이나 비즈니스 규칙을 공유함.

## 🛠️ 에이전트 작업 규칙 (Agent Rules)

### 1. 참조 우선순위 (Priority)
새로운 기능을 구현하거나 버그를 수정할 때, 에이전트는 아래 순서대로 정보를 참조합니다.
1. `README.md` (전체 프로젝트 지침)
2. **`.ai-lab/scripts/`** (검증된 로직 파일)
3. `.ai-lab/prompts/` (성공적인 수행 이력이 있는 프롬프트)
4. `.ai-lab/data-samples/` (로직 검증용 데이터 샘플)
5. 외부 검색 (최후의 수단, 반드시 사용자 동의 필요)

### 2. 코드 적용 가이드
- `.ai-lab/scripts/`에 있는 코드는 특정 기능을 위해 최적화된 결과물입니다. 
- 에이전트는 이 코드를 실제 서비스 코드(`public/`, `server.js` 등)에 이식할 때, 기존 프로젝트의 스타일(Naming Convention, Error Handling 등)을 유지하며 통합해야 합니다.

### 3. 피드백 루프
- 만약 `.ai-lab`의 로직에 오류가 있거나 개선이 필요하다고 판단되면, 직접 수정하기 전에 사용자에게 먼저 보고하고 수정을 제안해야 합니다.

## 📂 디렉토리 구조
- `scripts/`: AI Studio에서 생성된 JavaScript 함수 및 유틸리티 보관
- `prompts/`: 복잡한 태스크 해결에 사용된 시스템 프롬프트 및 페르소나 설정 기록
- `data-samples/`: 로직 검증에 사용된 공공데이터(JSON, CSV) 조각들

---
**Note**: 이 문서는 프로젝트의 '진실의 원천(Source of Truth)' 중 하나입니다. 지침을 위반하여 발생하는 불필요한 토큰 낭비는 엄격히 금지됩니다.
