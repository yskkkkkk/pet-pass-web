# 📜 Pet-Pass Development History

이 문서는 Pet-Pass 프로젝트의 개발 이력을 기록하고 관리하기 위한 공간입니다. 모든 AI 도구는 주요 변경 사항이 발생할 때마다 이 파일의 **최상단**에 작업 내역을 추가해야 합니다.

## 🛠️ 관리 규칙
1. **기록 시점**: 큰 기능 업데이트, 버그 수정, 아키텍처 변경이 완료된 직후 기록합니다.
2. **작성 위치**: 항상 최신 작업 내용이 가장 위(최상단)에 오도록 추가합니다.
3. **형식 준수**: 아래의 템플릿 형식을 반드시 유지합니다.
4. **협업 안내**: 다른 도구(예: Antigravity)가 작업을 마칠 경우, 이 파일에 기록하도록 상호 안내합니다.

## [2026-04-17] - Jules
### ✅ 작업 요약
- 모바일 퍼스트 전환 및 사용자 경험(UX) 최적화 작업 수행

### 🔧 변경 사항
- **뷰포트 최적화**: 모바일 브라우저의 가변적인 뷰포트 높이 문제를 해결하기 위해 `--vh` 계산 로직 도입 (`public/script.js`)
- **GPS 기능 추가**: `navigator.geolocation`을 이용한 '내 위치' 찾기 기능 및 버튼(#btn-my-location) 구현
- **필터 동기화**: 데스크탑과 모바일 검색바의 상태를 일치시키고 전체 초기화를 수행하는 `resetAllFilters` 로직 강화
- **성능 및 시각적 피드백**: 스토어 목록 로딩 시 Skeleton UI 적용 및 폰트를 'Pretendard Variable'로 교체하여 가독성 향상
- **UI/UX 개선**: 터치 타겟 크기 최적화(44x44px 이상) 및 모바일 전용 리셋 버튼(#btn-reset-mobile) 추가

### 📌 비고 (Issues & Decisions)
- **모바일 레이아웃**: `100vh` 사용 시 주소창에 의해 하단이 가려지는 현상을 방지하고자 JS로 계산된 `--vh` 값을 CSS 변수로 할당하여 사용함.
- **데이터 로딩**: 1,500개 이상의 매장 데이터를 처리할 때 발생하는 UI 프리징을 방지하기 위해 렌더링 최적화 및 스켈레톤 UI를 통한 체감 속도 개선.
- **사용자 위치 권한**: 권한 거부 시 기본 좌표로 부드럽게 복구되도록 예외 처리 로직 적용.
