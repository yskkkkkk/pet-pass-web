# 📜 Pet-Pass Development History

이 문서는 Pet-Pass 프로젝트의 개발 이력을 기록하고 관리하기 위한 공간입니다. 모든 AI 도구는 주요 변경 사항이 발생할 때마다 이 파일의 **최상단**에 작업 내역을 추가해야 합니다.

### 📌 비고 (Issues & Decisions)
- **기능 단순화**: 서비스 운영 방향에 따라 사용자 직접 등록 제안 기능을 제거하고, 공식 데이터 중심으로 서비스를 간소화함.
- **의존성 정리**: 더 이상 사용하지 않는 Supabase 관련 환경 변수 및 설정 정보를 삭제하여 프로젝트 복잡도를 낮춤.

## 🛠️ 관리 규칙
1. **기록 시점**: 큰 기능 업데이트, 버그 수정, 아키텍처 변경이 완료된 직후 기록합니다.
2. **작성 위치**: 항상 최신 작업 내용이 가장 위(최상단)에 오도록 추가합니다.
3. **형식 준수**: 아래의 템플릿 형식을 반드시 유지합니다.
4. **협업 안내**: 다른 도구(예: Antigravity)가 작업을 마칠 경우, 이 파일에 기록하도록 상호 안내합니다.

### ✅ 작업 요약
- 바텀 시트 인터랙션 최적화 및 [내 위치] 버튼 UX 동기화 개선

### 🔧 변경 사항
- **드래그 중 버튼 숨김 정책 적용**: 시트 이동 시 [내 위치] 버튼이 지연되어 따라오는 현상을 방지하기 위해 드래그 중에는 버튼을 즉시 숨기고(`opacity: 0`), 안착 후 다시 나타나도록 개선 (`public/script.js`, `public/style.css`)
- **MAX 상태 탈출 로직 강화**: 바텀 시트가 최대 확장(`85vh`) 상태에서도 상단 헤더나 핸들을 잡으면 내부 스크롤 상태와 관계없이 항상 드래그가 시작되도록 판정 로직 수정
- **터치 영역 확장**: `drag-handle`의 물리적 디자인은 유지하되, 가상 요소를 통해 실제 터치 가능한 영역을 대폭 확장하여 조작 편의성 향상
- **CSS 셀렉터 최적화**: 드래그 중 불필요한 트랜지션을 차단하기 위한 셀렉터 버그 수정 및 성능 최적화

### 📌 비고 (Issues & Decisions)
- **자연스러운 인터랙션**: 버튼이 실시간으로 따라오는 방식보다 조작 시 숨기는 방식이 모바일 환경에서 더 정갈하고 반응 속도가 빠르게 느껴짐을 확인하여 적용함.
- **UX 병목 제거**: 시트 상단 영역의 터치 우선순위를 높여 사용자가 '멈췄다'고 느끼는 조작 고착 현상을 해결함.

## [2026-04-17] - Antigravity
### ✅ 작업 요약
- 가변형 3단계 바텀 시트(Bottom Sheet) 모바일 레이아웃 구현 및 UI 고도화

### 🔧 변경 사항
- **3단계 가변 바텀 시트**: 모바일 기기에서의 효율적인 목록 탐색을 위해 3단 스냅(MIN:140px, MID:50vh, MAX:85vh) 기능 및 터치 드래그 로직 구현 (`public/script.js`, `public/style.css`)
- **레이아웃 오버레이 구조화**: 시트 높이 변화에 따라 지도가 찌그러지는 현상을 방지하기 위해 지도를 100% 고정하고 시트를 그 위에 Absolute로 띄우는 오버레이 아키텍처 도입
- **UI 동적 연동**: 바텀 시트의 높이(`--sheet-height`)에 맞춰 플로팅 버튼('이 지역 탐색', '내 위치', '맨 위로')의 위치가 부드럽게 연동되도록 CSS Variable 기반 인터랙션 적용
- **데이터 검색 연동**: 필터 초기화 시 바텀 시트 높이를 자동으로 MID 스팟으로 복원하는 `resetBottomSheet` 로직 추가

### 📌 비고 (Issues & Decisions)
- **레이아웃 안정성**: 기존의 Flex 기반 레이아웃을 Absolute 기반으로 전환하여 바텀 시트를 끌어올릴 때 카카오 맵의 마커 렌더링이 재계산되거나 깨지는 현상을 해결함.
- **성능 최적화**: 드래그 중 브라우저 리페인팅 부하를 줄이기 위해 `throttle`을 적용하고, 드래그 상태(`dragging`) 클래스를 통해 불필요한 트랜지션을 일시 해제함.

- **데이터 로딩**: 1,500개 이상의 매장 데이터를 처리할 때 발생하는 UI 프리징을 방지하기 위해 렌더링 최적화 및 스켈레톤 UI를 통한 체감 속도 개선.
- **사용자 위치 권한**: 권한 거부 시 기본 좌표로 부드럽게 복구되도록 예외 처리 로직 적용.
