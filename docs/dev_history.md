- **조회 파라미터 최적화**: 정부 API(v3) 호출 시 데이터 미반환의 원인이 되었던 `owner_nm=%20` 파라미터를 제거하고, `_type=json` 파라미터를 추가하여 응답 일관성 확보.

- **백엔드 로직 동기화**: Vercel Serverless Function(`api/auth-pet.js`)과 로컬 개발 서버(`server.js`)의 조회 로직을 동일하게 업데이트하여 환경에 구애받지 않는 정상 동작 보장.
- **방어적 파싱 구현**: API 서버의 상태에 따라 JSON 대신 XML이 반환될 수 있는 케이스를 고려하여, 응답 데이터 타입에 따른 가변적 파싱 로직 유지 및 강화.

### 📌 비고 (Issues & Decisions)
- **보안**: API 키는 코드에 하드코딩하지 않고 환경 변수(`DATA_GO_KR_API_KEY`)를 통해 주입받도록 구성하여 보안성 유지.
- **데이터 부재 대응**: 유효한 번호임에도 정부 API에서 데이터를 반환하지 않는 특수한 경우(개인정보 미동의 등)를 대비해 기존의 테스트 데이터 폴백 로직을 유지하여 사용자 경험 저하 방지.
